Welcome to the documentation page!

The purpose of this project is to build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies.

Request URL HTTP Method Request data format Response data format
Get a list of all movies /movies GET None A JSON object holding data about all the movies
Get data about a single movie by title /movies/[Title] GET None A JSON object holding data about a single movie
Example:
{
"Series": {
"Name": "Marvel Cinematic Universe Films",
"Description": "The Marvel Cinematic Universe (MCU) films are a series of American superhero films produced by Marvel Studios starting in 2007 based on characters that appear in publications by Marvel Comics. The MCU is the shared universe in which all of the films are set."
},
"Director": {
"Name": "Ryan Fleck",
"Bio": "Ryan K. Fleck is an American film director, cinematographer, editor, and screenwriter.",
"Birth": "1976"
},
"_id": "6158cd372e1bb8fa86ab6e58",
"Title": "Captain Marvel",
"Description": "Origin story of Captain Marvel",
"ImagePath": "captainmarvel.png",
"Featured": false
}
Get data about a director /movies/directors/[Name] GET None A JSON object holding data about a director containing bio, birth year and death year.
Example:
{
bio: "Joe Johnston is an American film director known for his special effects.",
birthYear: 1950,
deathYear: N/A
}
Allow new users to register /users POST A JSON object holding data containing id, first name, last name, and email
Example:
{
"Username": "CaptainSteve",
"Password": "firstcap",
"Email": "captainsteve@mail.com",
"Birthday": "1918-07-04T00:00:00.000Z",
} 		A verification message if the registration has been successful or not.
Allow users to update their user data by id /users/:id PUT A JSON object holding data containing the user data to be changed.
Example:
{
"Username": "dianaPrince",
"Password": "wonderwoman",
"Email": "shouldbemarvel@mail.com"
} 		A verification message that a new username was successfully assigned or failed.
Allow users to add a movie to their list of favorites /users/:userName/favorites PUT A JSON object holding the user information with the name of the movie to be added.
Example:
{
"_id": "6164a606454e189100fa166e",
"Username": "dianaPrince",
"Password": "wonderwoman",
"Email": "shouldbemarvel@mail.com",
"FavoriteMovies": [
"6158cd372e1bb8fa86ab6e58"
],
"__v": 0
} 		A message whether the film has been added successfully to favorites or not.
Allow users to remove a movie from their list of favorites /users/:userName/favorites DELETE A JSON object holding the user information with the name of the film removed
Example:
{
"_id": "6164a606454e189100fa166e",
"Username": "dianaPrince",
"Password": "wonderwoman",
"Email": "shouldbemarvel@mail.com",
"FavoriteMovies": [],
"__v": 0
} 		A message whether the film has been removed successfully or not from the favorites.
Allow users to deregister by username /users/:userName DELETE The username of the user to be deleted should be put after /users/ in the URL. A verification message if the deregistration has been successful or not.
